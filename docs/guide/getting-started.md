# What is aglang?

**aglang** (Architecture Ground Language) is an agent-first DSL for enforcing architectural rules at git-commit time using formal verification (Z3 SMT solving).

Unlike linters that check code style, aglang checks **architectural intent** — the relationships between components, data flows, and security boundaries you actually care about.

## How it works

```
Your codebase                   Your .ag spec
     │                               │
     ▼                               ▼
[Git Diff Extractor]    +    [Architecture Compiler]
     │                               │
     └──────────── Z3 Solver ────────┘
                        │
               UNSAT → commit passes ✓
               SAT   → commit rejected with proof ✗
```

1. **You write** an `architecture.ag` file describing components and invariants
2. **`aglc compile`** translates it to SMT-LIB and emits `architecture.o`
3. **A pre-commit hook** extracts the staged diff, runs Z3, and blocks violations

## Designed for AI Agents

AI coding agents (Copilot, Claude, Cursor, etc.) can operate autonomously for long stretches. aglang acts as **guardrails** — mathematical constraints the agent cannot violate even if it hallucinates an architecture change.

- `aglc add` generates a starter spec, installs the hook, and emits `skill.json` in one command
- Agents read `AGENTS.md` (generated by `aglc emit-context`) to understand boundaries before they start
- Every commit attempt is checked — human or agent alike

## Installation

::: code-group

```bash [npm (global)]
npm install -g @collivity/aglang
```

```bash [npx (no install)]
npx @collivity/aglang compile architecture.ag
```

```bash [GitHub URL]
npm install github:collivity/aglang
```

:::

## Quick start

```bash
# One-shot setup for any project
aglc add /path/to/your/project --name MyApp

# That's it — the spec, artifact, hook, and skill.json are ready
# Open architecture.ag and add invariants
```

See [Quick Setup (aglc add)](./generate) for details.

## Requirements

- **Node.js** ≥ 18
- **Z3** — install with `pip install z3-solver` (Python) or `brew install z3` (macOS)
- Git repository (for hook installation)
